# Chapters v5.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the chapter framework plus two playable chapters: The Body (current game re-framed, 2 new enemy behaviors) and The Pond (currents, obstacles, new roster, 2 new weapons).

**Architecture:** Chapters are pure data in `CHAPTERS` (config.js); sim stays theme-agnostic and reads behavior flags/stat multipliers from the run's chapter snapshot. Meta gains per-chapter ladders (`meta.chapters[id]`) with a one-time migration. UI gains a title-screen chapter selector; render gains per-chapter palettes/props/skins.

**Tech Stack:** PixiJS v8, Vite 6, plain JS ESM, custom test harness (`npm test`, runs A–S in test/sim-test.js).

Spec: `docs/superpowers/specs/2026-07-16-chapters-design.md` — read it first.

## Global Constraints

- Plain JS ESM, 2-space indent, **no semicolons**.
- `src/sim.js` is pure simulation — no Pixi, no DOM. `src/config.js` owns ALL tunable numbers. `src/state.js` doc comments are the authoritative shape contracts — update them with every shape change.
- Weapon-mod parity rule (v4.3): every weapon has 6 mods including ≥1 behavioral.
- Art: real sprite assets only (Kenney CC0 in `src/props`, `src/fx`) — no bare procedural Graphics shapes. Reuse/tint existing sprites where new ones aren't available.
- Never break existing saves: migration must be lossless for coins/shop/choiceSlots/best.
- After every task: `npm test` passes, `npm run build` succeeds. Commit per task (no Claude signature lines in commit messages).
- Weapon ids in code today: `star, orbit, wave, boomerang, mines, homing, hole, rainbow`. In v5.0, `boomerang`, `hole`, `rainbow` belong to unshipped chapters and thus leave the pool (intentional, per spec).

### Task 3: Sim — behavior flags, currents, obstacles

**Files:**
- Modify: `src/sim.js`, `src/config.js` (flag tunables), `src/state.js` (doc comments: run.obstacles, enemy flag fields)
- Test: `test/sim-test.js` (new run V)

**Interfaces:**
- Consumes: run.chapter, CHAPTERS roster/eliteFlags/signature/obstacles from Tasks 1–2.
- Produces (all tunables in config.js):
  - Roster application: spawnEnemy picks a roster entry per archetype (skins/names for render/HUD later), applies hpMul/speedMul, sets `e.flags` (array) + `e.rosterId`.
  - `latch`: on contact with player, applies `run.player.slowT = LATCH_SLOW_T` (0.9s, LATCH_SLOW_MUL 0.55 to move speed) then the enemy dies (antibodies spend themselves).
  - `split`: on death (non-split children only) spawn 2 children at 0.45 hp/0.7 radius with `e._splitChild = true`; reuse the splitter-affix code path, generalized.
  - `dashBurst`: cycle — DASH_IDLE_T (1.1s) at 0.4× speed, then DASH_T (0.5s) at 2.6× toward player.
  - `acidPool` (elite flag): death leaves `run.pools` entry `{x, y, r: ACID_R 70, t: ACID_DUR 3, dps: ACID_DPS 8}` damaging the player standing in it (flag events `dot: true`).
  - `soapTrail` (elite flag): drops trail nodes every SOAP_INTERVAL 0.35s: `{x, y, r: 26, t: SOAP_DUR 2.5, dps: SOAP_DPS 6}` into the same `run.pools` array.
  - `currents` signature: `stepCurrents(run, dt)` — smooth vector field `F(x,y,t)` from 2 summed sine pairs (seeded per run, params from chapter signature config), force added to player and enemy positions each frame (not velocity-stored; drift, no control loss).
  - Obstacles: `createRun` generates `run.obstacles = [{x, y, r}]` per chapter config (rejection-sample min spacing, none within minDist of origin). Circle push-out for player and enemies in their movement steps. Projectiles unaffected.
  - `run.pools` documented in state.js; render events not required (render reads live arrays).
- Existing elite affixes remain and stack with eliteFlags.

**Steps:**

### Task 4: New weapons — Flagella Whip + Toxin Bloom (+ mines re-theme)

**Files:**
- Modify: `src/config.js` (WEAPONS entries, WEAPON_MODS entries, tunables), `src/sim.js` (weapon step functions + mod hooks), `src/state.js` (doc comments: run.blooms, whip fields)
- Test: `test/sim-test.js` (new run W)

**Interfaces:**
- Consumes: chapter pools from Task 2 (ids `flagella`, `bloom` already referenced there).
- Produces:
  - `flagella` (pond starter, rarity normal): melee arc lash. Every `rate` seconds sweeps an arc (`arc` rad, `range` px) centered on the player's facing (last-move or nearest-enemy direction), damaging every enemy whose center is inside the sector. Level scaling like other weapons (dmg/rate/range up). Pushes event `{type:'hit'}` per enemy struck; add `{type:'whip', x, y, angle, range, arc}` event for render.
  - `bloom` (rarity rare): every `rate` s plants a toxin cloud at the position of a random enemy within `castRange` (fallback: random offset near player): `run.blooms` entry `{x, y, r: 0, maxR, t: 0, dur, dmgPerTick}`; expands to maxR over dur×0.35 then ticks damage (dot-flagged hits) every 0.5s to enemies inside until expiry.
  - Mods (6 each, ≥1 behavioral, WEAPON_MOD structure identical to existing weapons):
    - flagella: `reach` (+35% range), `wideArc` (+30% arc), `frenzy` (+25% attack speed), `heavyLash` (+40% dmg), behavioral `cyclone` (every 3rd swing is 360°), behavioral `barbed` (struck enemies bleed 3×dmg over 3s, dot).
    - bloom: `bigBloom` (+35% maxR), `lasting` (+40% dur), `virulent` (+35% dmg), `quickCast` (+25% cast rate), behavioral `twinBloom` (plants 2 clouds), behavioral `sporeburst` (enemies dying inside a bloom emit a mini-bloom r=maxR×0.35, no chaining off mini-blooms).
  - mines re-theme: WEAPONS.mines name/desc/icon become 'Toxin Cysts' (behavior unchanged) — copy only.
- Balance bands (assert in tests, mirroring run P's style): flagella level-6 with 2 mods kills a standard ring slower than 3.5× the pond-median; bloom dot events must carry `dot: true` (no strobe/sfx).
